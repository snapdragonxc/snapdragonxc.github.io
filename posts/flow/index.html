Flow Type template for packages

September 2, 2018

Flow is the replacement for Prop-Types in ReactJS. When you install packages in your project, you will need the type definitions for these packages. This is a short post describing how to get these.

The type-checking of the interfaces of functions and classes in Javascript is regarded as good practice as it ensures more robust code. Flow is React’s solution to type-checking and is a direct competitor to Typescript. Compared to Typescript it has the convenience of adding another library to your package manager, rather than adding another compiled language. Therefore you can keep all your old settings that you have in your original Babel configuration.

In order to use type-checking, you will need to know the interface types of the packages that you install alongside React. The React-Router package is widely used and a good example. React Router passes params for the web-history as props to the child routes. Therefore for all your routes you are going to have to define the prop-types of the web-history (unless you want to ignore them).

Fortunately, Flow have thought of this and created a library of type-definitions called ‘flow-typed’ that you can install in your project:

https://flow.org/en/docs/libdefs/

This post shows how to install and use this library.

Flow-typed uses a command-line-interface (CLI) for pulling the type definitions of packages that you require into a bin that it creates on your desktop computer. You let your project have access to this bin by installing ‘flow-bin’ in your project.

To install the flow-typed CLI, from a terminal:

sudo npm install flow-typed -g

The install the flow type definition of the package(s) that you need, for example React-Router:

flow-typed install react-router

From a terminal inside your project folder install, flow-bin:

npm install --save-dev flow-bin

Inside the component file in your project where you need the type definitions, add the import to the top of your file:

import type { RouterHistory, Location, Match } from 'react-router'

If you get stuck you can look in the flow-typed repository, showing a list of available packages:

https://github.com/flow-typed/flow-typed/tree/master/definitions/npm

and within each folder, you get the type definitions:

https://github.com/flow-typed/flow-typed/blob/master/definitions/npm/react-router_v4.x.x/flow_v0.63.x-/react-router_v4.x.x.js
