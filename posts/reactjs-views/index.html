ReactJS Separating state from view

September 5, 2018

When working on large projects in ReactJS there are certain design patterns to follow which will make your life easier. This post describes the Container Component pattern by way of example.

ReactJS has two types of components. A dummy component, which does nothing but render HTML to the screen and a component with state, that renders HTML whenever it’s state is changed.

The dummy components are functions that return HTML in the form of a jsx template. For example the component below returns a heading with a name. The name is entered as a prop to the function. Most of the time you use functions for dummy components. You can use a class but the extra overhead of a class is unwarranted.

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

Lets now look at an example of a component with state. A good example is a form, which contains a text field for you to enter your name. The jsx for the component is the form itself, which is returned by the ‘render’ method.

class NameForm extends React.Component {
  constructor(props) {
    super(props);
    this.state = {value: ''};
  }

  handleChange(event) {
    this.setState({value: event.target.value});
  }

  render() {
    return (
      <form>
        <label>Name:</label>
        <input
          type="text"
          value={this.state.value}
          onChange={() => this.handleChange}
        />
      </form>
    );
  }
}

The state is the value of the text input. When this is changed via the keyboard, the ‘handleChange’ method is called and the state is updated with the ‘setState’ method.

Everytime the ‘setState’ method is called, the component will re-render the form with the new state value. The initial state is defined in the constructor.

With only a small amount of HTML to render, it is acceptable to combine the logic of a state component with the jsx. But when you have large amounts of HTML it quickly becomes a confusing clutter of code mixed up with jsx.

Lets now look at how we can re-factor the form using the Container Component pattern. We first define a dummy component, which I will call the ‘View’, which contains the jsx.

// View.jsx
function View(props) {
  return (
    <form>
      <label>Name:</label>
      <input
        type="text"
        value={props.value}
        onChange={() => props.handleChange}
      />
    </form>
  );
}
export.default View;

We then re-write the form as the ‘ViewContainer’ using the jsx of the View component as an interface as follows:

import View from './View.jsx';

class ViewContainer extends React.Component {
  constructor(props) {
    super(props);
    this.state = {value: ''};
  }

  handleChange(event) {
    this.setState({value: event.target.value});
  }

  render() {
    return (
      <View
        value={this.state.value}
        onChange={() => this.handleChange}
      />
    );
  }
}

This then is the Component Container pattern. With this pattern you can quickly and easily port your HTML into a view component with the corresponding logic in the container component.
