Redux When and where to use it

September 3, 2018

Redux manages the overall state of a ReactJS application with multiple components. The question is when to use it in your application and what components need to use it. This post provides two examples that help answer that question.

Redux is React’s version of a Model-View-Controller. The store is the model, the reducer is the controller and the components are the views. Components subscribe to the store so that when the store updates, they receive updated properties from the store. Components also issue actions to the reducer when they want to update the store. The flow is uni-directional.

Redux adds significant amount of code to your application so it requires careful planning beforehand. The following two examples show how it can be used to simplify code and reduce calls to the server.

Shopping Cart

A shopping cart is widely used in Ecommerce shops. Often we want to show a cart in the Header component which is visible on every page of the application and lets the user know how many items they have in the cart. When we update the Shopping Cart from a page in the application, we would also like to update the cart in the header at the same time. This is an example of where Redux considerably simplifes the code. Both the shopping cart and the header component subscribe to the store. Then when the cart updates the store, the header will be notified of the change in store and update accordingly.

Ecommerce Shop

Ecommerce shops display a list of products to buyers for them to select which product they would like to buy. When they select a product they are shown the details of the product they selected on another page. When the products page loads, a call is made to the server to get the products. Without Redux, we would have to make another call to the server when a product was selected.

With Redux, both the products page and the product page subscribe to the store. When the products page loads it issues an action to Redux which puts the products returned by the server into the store. Then when a buyer selects a product this is retrieved from the store without an additional call to the server being made.
